tachidesk部署指南

我愿称之为喜欢漫画的完美终结者，就是这么完美。Tachidesk是一个独立Tachiyomi的兼容软件，不是Tachiyomi的Fork。Tachidesk是一个免费的、开源的漫画阅读器服务器，可以运行为Tachiyomi构建的扩展。
如果你不知道Tachiyomi的话，你就不会知道这个软件的强大之处，Tachiyomi是一个通过插件查看第三方漫画网站的工具，和之前大火的的多多猫类似的功能，不过区别在于它更加稳定，而且目前开源有人维护。
而Tachidesk则是其开源的桌面版本，支持windows，Linux，macOS等主流系统。
优势：
- 开源，且每周更新，更新强度不低
- 可以查看主流和非主流的所有漫画网站
缺点：
- 某些特殊漫画网站需要翻墙的话，你也的翻墙。
- 没有用户登陆机制，所以部署在外网需要手动增加基础认证

部署

首先说明部署，部署的话我采用的是docker方式，根据他的docker说明，它的docker房子位置不在docker.io，这和很多其他的地方不一样，所以需要使用：

docker pull ghcr.io/suwayomi/tachidesk:preview-alpine

支持多个版本

然后执行：

docker run -p 4567:4567 ghcr.io/suwayomi/tachidesk

就可以在127.0.0.1:4567上访问。
界面：

  • 主页面
  • 插件页面

    插件安装记得筛选中文
  • 设置页面

    虽然说页面不提供翻译，但是其实都很简单，不行可以下载Tachiyomi的界面进行对照，两边界面完全一样。

总结

我觉得这个聚合漫画的方式可以完全满足我看漫画的需求，也不需要去自己繁琐的下载裁剪，可以通过网页直接查找自己需要的漫画：推荐57漫画这个插件，里面的东西大多数都有，而且还新。
